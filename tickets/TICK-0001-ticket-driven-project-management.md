# Title

Establish ticket-driven project management workflow

## Status

done

## Priority

high

## Type

process

## Summary

Define a repository-wide workflow where every requirement must be recorded as a standalone markdown ticket before implementation begins.

## Scope

- Create `AGENTS.md` with the ticket-only workflow rules
- Define where tickets live and how they are structured
- Start recording current known requirements as markdown tickets

## Out of Scope

- Issue tracker integration
- Automation for syncing tickets to external systems

## Acceptance Criteria

- `AGENTS.md` exists and documents the ticket-only workflow
- `tickets/` exists and contains current known requirements
- The workflow explicitly forbids implementing undocumented requests

## Notes

- This ticket establishes the process baseline for all future work in the repository.
