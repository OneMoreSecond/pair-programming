# Title

Add structured review export and protocol evolution support

## Status

todo

## Priority

low

## Type

feature

## Summary

Evolve the review protocol so the workflow can expose more machine-readable data while preserving markdown usability and backward compatibility.

## Scope

- Define how protocol versions evolve over time
- Add a more structured export format for review results when needed
- Preserve compatibility with existing markdown-based review artifacts

## Out of Scope

- Replacing human-readable markdown entirely
- Designing a full external API surface

## Acceptance Criteria

- The workflow has a documented path for protocol evolution
- Review results can be consumed in a more structured form when needed
- Existing markdown artifacts remain valid or migratable

## Notes

- This requirement comes from the protocol and architecture design docs.
